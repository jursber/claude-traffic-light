# VibeLight 呼吸与 PWM 调试 — 问题与经验（固化备忘）

本文记录 **VibeLight / traffic_light.ino** 在呼吸观感、USB 调试、并发与刷新路径上踩过的坑与最终结论，便于以后改固件或排障时少试错。操作级清单仍以 **[TUNING.md](TUNING.md)** 为准。

---

## 1. 现象：呼吸「平顶」、长时间像顶在最亮、或明暗节奏怪

### 根因 A：PROTO 灯效跟 `loop()` 刷新绑在一起

主循环里 **NimBLE + USB 串口解析** 可能长时间占 CPU，若 PWM/包络只在 `loop()` 里更新，会出现 **占空比长时间不变** → 肉眼像 **卡在峰值亮度**。

**经验**：PROTO 下的灯效（尤其呼吸）应由 **独立 FreeRTOS 任务** 以固定节拍（约 **1 ms**）调用 `updateProtoLeds()`，与 `loop()` 解耦。任务优先级须 **高于** Arduino `loopTask`（约 1），且 **低于** NimBLE 主机任务；默认宏 **`TL_PWM_TASK_PRIO`**（如 8）可按板子微调。

### 根因 B：相位用 `micros()` 在射频忙时异常

`micros()` 在部分射频栈繁忙场景下可能出现 **停顿/跳变**，呼吸相位会 **卡在包络峰值附近** → 「长时间最亮 + 极短暗」。

**经验**：相位统一用 **`esp_timer_get_time()`**（64 位微秒单调）计算包络。

### 根因 C：三角波两半周期除法不对称

若上升、下降两段用不同形式的除法/取整，会造成 **数学上** 的轻微不对称。

**经验**：用单式 **`x = ph * 510 / span`** 再折返 **`base = x` 或 `510-x`**，保证两半对称（在仍为线性三角的前提下）。

---

## 2. 现象：只插 USB、没连蓝牙，问题仍在

**结论**：`setup()` 里仍会 **`initBle()`**，NimBLE **照样占 CPU/RAM**。「不测 BLE」不等于「没有 BLE 负载」。

**经验**：若要做 **纯串口对比实验**，可临时注释 `initBle()` 重编（注意 NimBLE 相关符号是否仍需 `#if` 包一层，依 Arduino-ESP32 版本而定）。

---

## 3. 现象：GUI 发 PROTO 呼吸正常，过一会儿又变常亮 / 不再呼吸

### 根因：同一 COM 上 ASCII 把引擎切回 LEGACY

固件兼容旧协议：**单字节 `G/g/y/Y/r/R/O`** 会 **`applyLegacyCommand()`** 并把 **`s_engine` 设为 `ENGINE_LEGACY`**。此后 PWM 任务 **不再** 调 `updateProtoLeds()`，PROTO 呼吸停止；若 Legacy 为某路 **255 常亮**，观感即「一直亮」。

本仓库 **Claude/Codex 红绿灯守护进程** 默认向 **同一 USB 串口** 写这些字符。

**经验**：用 **`vibelight_gui.py`** 做 PROTO/呼吸实验前，**务必停掉** 会向该 COM 写 ASCII 的进程（统一守护进程、第二个 `set_state`、误开的串口助手自动发码等）。详见 **TUNING.md §1**。

---

## 4. 现象：数学上三角波对称，但「暗→亮」仍明显不对称

线性 **PWM 占空比** 对人眼 **非线性**：低占空比区间「加一点就亮很多」，高段要爬很久才觉得「到顶」。体感常像 **暗→很快到八成观感、末段慢慢顶满**；而 **亮→暗** 有时反而更接近主观上的「正常」。

**经验（三角模式）**：对 **`MODE_BREATH` 仅上升半周** 做整数 **`(lin²+127)/255`**，压低前段斜率、后段更陡；**下降半周保持线性**，避免破坏已满意的「亮→暗」。由宏 **`TL_BREATH_RISE_PERCEPTUAL`** 控制（默认 1；示波器要严格对称三角可置 0）。

## 5. 其它工程细节

| 点 | 说明 |
|----|------|
| **并发** | `tryApply*` 与 `updateProtoLeds` 对共享灯态用 **`portMUX_TYPE` 临界区**；刷新路径先 **快照** 再算 PWM，缩短持锁时间。 |
| **环缓解析** | BLE 回调与 `loop` 都往环缓推字节；解析避免在 NimBLE 回调里长时间持锁。 |
| **USB 读爆** | `while (Serial.available())` 一次读光可能拖长单圈 `loop`；用 **`TL_SERIAL_DRAIN_MAX`** 限制每圈最大读取字节数。 |
| **扩展帧节流** | **`PROTO_MIN_INTERVAL_MS`（4 ms）** 防写风暴；调参时以「单帧或低速发」为准。 |
| **正弦类包络** | 曾内置 **`sin(π·t/T)`** LUT，实测仍易出现与「线性 PWM + 人眼」类似的观感问题，已 **从固件移除**；默认 **`MODE_BREATH`** 恢复为 **三角 + 上升 `lin²` 整形**。需要正弦或其它形状请发 **`cmd=2` 自定义曲线**。 |
